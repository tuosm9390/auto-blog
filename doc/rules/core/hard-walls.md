# 🚨 절대 금기 사항 및 보안 수칙

## 1. 보안 지침 (Security Mandates)

- **IDOR 방어**: 모든 리소스 접근(API) 시 소유권 검증 로직(eq("author", username))을 필수로 포함한다.
- **RLS 준수**: 데이터베이스 정책에서 USING (true)를 절대 사용하지 않는다.
- **시크릿 관리**: API 키, DB 자격 증명 등을 절대 하드코딩하지 않으며 .env를 통해서만 관리한다.
- **Zero Trust**: 클라이언트 데이터를 신뢰하지 않으며, Zod를 통해 모든 입력값을 검증한다.

## 2. 3-Strike 정책 (무한 루프 방지)

- 동일한 에러가 2회 발생 시 접근 방식을 즉시 변경한다.
- 3회 연속 실패 시 코드 수정을 중단하고 'Suspects' 목록을 작성하여 보고한 뒤 사용자의 지시를 기다린다.

## 3. 코드 삭제 금지

- 사용자의 명시적 요청 없이 기존의 안정적인 로직이나 주석을 대량 삭제하지 않는다.
