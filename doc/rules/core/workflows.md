# ⚙️ 작업 절차 및 워크플로우

## 1. 개발 주기

- **Research -> Strategy -> Execution (Plan-Act-Validate)** 순서를 엄격히 준수한다.
- 코드 수정 전 doc/results/에 분석 리포트 및 구현 계획서를 먼저 작성한다.

## 2. TDD (Test-Driven Development)

- 기능 구현 시 Red-Green-Refactor 사이클을 준수하며 테스트 코드를 동반한다.

## 3. Git 컨벤션

- feat:, fix:, refactor:, docs: 등의 접두사를 사용하며 'Why'를 설명하는 커밋 메시지를 작성한다.
